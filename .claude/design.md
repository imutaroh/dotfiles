# 寒色トーン&マナー（デザイン言語）

このドキュメントは、dotfiles のステータスライン（`statusline.sh`）とカスタムテーマ（`dark-daltonized-cool.json`）で実際に使っている「寒色ベースの配色・表現ルール」をまとめたものです。今後ほかの UI（HTML プレビュー、ダッシュボード等）で同じトーンを再利用するための共通リファレンスとして使います。記載した HEX/RGB はすべて実コードのコメント・定義に基づく実値です。

## コンセプト

- **寒色ベース（cool）**を基調にする。シアン・ブルー・ティール・バイオレット・スレートで全体を構成する。
- **暖色は原則使わない。** 唯一の例外は「危険（使用率 90% 超）」のときのコーラル 1 色だけ。アクセントとして節度を持って使う。
- 目指す印象は **落ち着いた・モダン・視認性重視**。装飾より情報の読み取りやすさを優先する。

## カラーパレット

Tailwind の 400 系（寒色）で統一している。トークン名は `statusline.sh` の変数名に合わせている。

| 名前（変数） | HEX | RGB | 用途（UI のどこの色か） |
| --- | --- | --- | --- |
| CYAN | `#38bdf8` | rgb(56,189,248) | モデル名 `[...]`（sky-400） |
| BLUE | `#60a5fa` | rgb(96,165,250) | ディレクトリパス（blue-400） |
| TEAL | `#2dd4bf` | rgb(45,212,191) | git ブランチ名 / プログレスバー通常時の塗り（teal-400） |
| VIOLET | `#818cf8` | rgb(129,140,248) | git の dirty マーカー `*`（indigo-400） |
| ICE | `#94a3b8` | rgb(148,163,184) | ラベル・区切り `\|`・補助情報（コスト/経過時間/リセット残り）（slate-400） |
| WARN | `#60a5fa` | rgb(96,165,250) | 使用率 70% 以上の警告（明るい青。BLUE と同値） |
| RED（コーラル） | `#f08a8a` | rgb(240,138,138) | 使用率 90% 以上の危険（輝度高め・彩度控えめのコーラル。唯一の暖色アクセント） |

> 補足: 危険色の変数名はコード上 `RED` だが、実際の色は純赤ではなく控えめなコーラル。本ドキュメントでは意図を明確にするため「コーラル」と併記する。

## 使用率による段階表現（escalation）

プログレスバーと数値は、使用率に応じて 3 段階で色を変える（`usage_color` 関数）。

| 使用率 | 色 | トークン |
| --- | --- | --- |
| 0–69% | 通常色（バー/値は TEAL） | TEAL |
| 70–89% | 明るい青 | WARN（`#60a5fa`） |
| 90%+ | コーラル | RED（`#f08a8a`） |

### 設計意図: なぜ純赤を避けるか

純赤（`#f85155` のような彩度 MAX の赤）は暗背景上でギラついて、かえって視認性が落ちる。そこで彩度を抑えて輝度を上げた控えめなコーラル（`#f08a8a`）を採用し、「危険であることは伝わるが、目が疲れない」バランスにしている。

## レイアウト原則

- **縦に積む（1 行 = 1 つの関心事）。** ステータスラインは最大 4 行（1: モデル+ディレクトリ+git / 2: ctx バー+コスト+経過時間 / 3: 5h レート制限 / 4: 7d レート制限）。ターミナルを分割して幅が狭くなっても、各情報が途中で切れずに見えるようにするため。
- **絵文字は使わない。** フォント依存の幅ズレや豆腐（文字化け）を避けるため。意味は「ラベル + 色 + バー」で表現するミニマル方針。
- **プログレスバーは 10 マス固定**（`make_bar`）。`▓`=使用 / `░`=空 で、塗りの色は上記の段階色を使う。
- **ラベルは 3 カラム幅に左揃え**（`ctx` / `5h` / `7d`）。縦のリズムを揃えて視線を流しやすくする。
- **補助情報は目立たせない。** リセット残り時間は `↻` 記号 + ICE（dim）。コスト・経過時間も区切り `|` とともに ICE で表示し、主役の寒色情報より一段沈める。

## アクセシビリティ

- ベーステーマに色覚配慮の **dark-daltonized** を継承したカスタムテーマ **dark-daltonized-cool**（`Dark Daltonized (Cool)`）を使用する。
- 危険の表現を **色相（赤）だけに頼らない。** 「輝度の高いコーラル」を使うことで、赤緑の判別が弱い人でも明暗で危険を認識しやすくしている。
- テーマでは `warning` トークン（auto モードの枠線・警告全般）を `#2dd4bf`（teal-400）に寒色化し、付随する `warningShimmer` も `#5eead4`（teal-300）に揃えている。これにより警告系の UI もトーンを崩さず寒色で統一される。

## 再利用ガイド

ほかの UI（HTML プレビュー等）で同じトーンを使うときの原則:

- **背景は暗色**（`#0a0a0a` または `#0d1117`）。
- **テキストの主役は寒色**（CYAN / BLUE / TEAL / VIOLET）。
- **補助情報はスレート**（ICE / slate-400）で一段沈める。
- **危険のみコーラル 1 色**（RED / `#f08a8a`）。それ以外で暖色を足さない。
- **状態は色相だけでなく明暗・記号でも伝える**（escalation の 3 段階、`▓/░`、`↻`、`*` など）。
- **絵文字に頼らず、ラベル + 色 + バーで意味を表す。**
