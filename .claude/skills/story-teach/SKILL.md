# Story Teach — 知識を「映画版×要点カード」の読み物に変える

## これは何か

リスト・辞書・リサーチ結果のような「正しいが平板な知識」を、**読者本人（二人称「君」）を主人公にした一本の物語**へ作り替える。情報を箇条書きで並べず、物語のヤマ場ごとに知識が「必要になって」登場する。`The Goal`（小説で経営理論を教える本）と同じ構造。

**二層設計が肝**：
- **物語レイヤー** … 通読すると「読み物」。映画の濃度（内心・間・表情・往復セリフ）で書く。
- **カードレイヤー** … 各幕の末尾に `要点カード` を集約。末尾に索引表。カードだけ拾えば「辞書」。

> なぜ二層か：網羅（収集・出典・両論併記）は良い"地図"を作るが、地図を物語に混ぜると平板化する。だから物語に網羅を**溶かさず、物理的に別レイヤー**へ逃がす。カードは**シーンの途中に挟まない**（没入が切れる）。必ず**幕の最後**に置く。

## いつ使わないか（境界）

- 個人の体験談を note 公開記事に → `note-create`
- 技術・コードを Zenn で解説 → `learn`
- 購入や行動を促すセールス本文 → `copywriting-frameworks`
- このスキルは「**知識を、読者を主人公にした物語で教える**」専用。学習・行動変容・自己理解が目的の長め記事に向く。

## ワークフロー（順に進める）

### 0. 素材と読者を集める

書き始める前に、最低この4つを確定する。曖昧なら読者に1〜2問だけ聞く。

1. **素材**：物語に畳み込む知識（リサーチ、辞書ノート、概念リスト）。←情報量はここで決まる
2. **読者は誰か**：「君」として登場させる具体人物。職業・痛み・Vision・強み・実際の生活シーン。**ここが薄いと「自分ごと」にならず必ず失敗する。** ローカルの user メモリ（プロフィール・動機・強み）があれば読む。
3. **主戦場（シーン）**：読者が実際に直面する2〜4個の具体場面。これが各「幕」になる。
4. **ゴール**：読み終えた読者に何が起きてほしいか（行動・視点の反転・スキル習得）。

### 1. 診断して反転する

まず「素材をそのまま出すと、なぜ退屈か」を読者に短く突きつけてから直すと、納得感が出る。平板化の4大要因（`references/craft.md` の診断表）を踏まえ、対になる解を用意する：リスト→物語／読者不在→読者を主人公に／主役多すぎ→1点突破×幕／科学盛りすぎ→ヒーロー1本。

### 2. 背骨（スパイン）を組む

- **主人公＝読者「君」**。二人称で、読者の実際の現場を描く。
- **一本の連続した弧**。バラバラの例示ではなく、君が「できなかった→できる」へ変わる物語。
- **各幕＝1つの主戦場**。幕ごとに1つの場面だけを扱い、そこで必要な手だけ出す。
- **テンポを幕ごとに変える**（長尺の中だるみ対策）。例：第1幕＝スローバーン、第2幕＝速いカット割り、第3幕＝短く温かく。
- **終幕で読者の隠れた動機に答える**（「本当に欲しかったのは○○だ」とゴール／Visionへ接続）。

### 3. ヒーローの事実を1つだけ選ぶ

- 物語の**転換点に置く科学/データを1本だけ**ドラマにする（例：自己開示＝脳の報酬）。これが「視点の反転」の燃料。
- 残りの裏付けは各シーンに**1行ずつ溶かす**（出典は括弧で軽く）。5本並べて温度を下げない。

### 4. 各幕を「映画の濃度」で書く

要約せず実時間で描く。詳細な技法（内心の声・間・"危ない、寸前で気づく"テンション・セリフのフィラー・五感の設定）は **`references/craft.md` を読んでから書く**。

### 5. 幕末にカードで結晶化する

各幕の最後に、その幕で主人公が使った手を `[!success]` の **要点カード** で畳む。全幕通しで**通し番号**を振り、末尾に**索引表**を置く（辞書として試合前に開ける）。

カードの型（Obsidian Callout。`[!success]` 固定）：
```markdown
> [!success] 第N幕で使った手
> **① カード名**：何をやるか（一言）。←なぜ効くか（出典）
> **② カード名**：…
```

### 6. Company/Drafts に出力する

ファイル名 `Company/Drafts/YYYY-MM-DD-slug.md`。frontmatter：
```yaml
---
type: ai-work
status: review
created: YYYY-MM-DD
topic: 一言で何の成果物か（映画版）
promote: Mind   # 昇格先の見当
---
```
大きいので一発 Write せず、骨子→各幕を Edit で継ぎ足す（ツール呼び出しパース失敗の予防）。元の辞書/リサーチノートがあれば相互リンクを張る。

## 品質バー（出す前に自己採点）

- 冒頭3行で、読者が「これは自分の話だ」と感じるか（読者本人＋具体シーン）。
- 各幕は1場面・1点突破か（主役が増えていないか）。
- 科学は転換点に1本か（盛りすぎていないか）。
- カードはシーン途中でなく**幕末**にあるか。索引はあるか。
- **縦スクロール組版になっているか**（1文1行・意味の塊でブロック化・空行区切り。横長パラグラフは流し見できず失格）。詳細は `references/craft.md` 4.5。
- 終幕に「持ち帰り（行動 or 視点反転）」と「明日やる1個」があるか。
- 媒体の住み分けを侵していないか（体験談/技術/セールスは別スキル）。

## 失敗モード

- **読者が一般人化**：「君」が抽象的だと刺さらない。実在の痛み・強み・生活を入れる。
- **どん底長尺**：全幕を同じ濃度で盛ると中だるみ。テンポの緩急で守る。
- **カード挟み込み**：没入が切れる。必ず幕末へ。
- **網羅の誘惑**：素材を全部入れたくなる。カード化して畳み、物語本文は1点突破を死守。
