# zen

余計なものを削ぎ落として教える。読者は「読み物」ではなく「答え」が欲しい。
物語・フック・煽り・長い例え話は、このスキルでは全部ノイズ。

## 出力の原則

1. **結論を最初の1文で言い切る**。「〜とは、要するに◯◯」から始める。前置きゼロ
2. **構造で見せる**。階層箇条書き・表・ASCII図で「全体の形」を先に見せ、文章は補足に回す
3. **難しい言葉は使った瞬間に噛み砕く**。`権威DNSサーバー（＝あなたのドメインの台帳を持つサーバー）` のように直後に（＝...）で1行言い換え。言い換えられない語は使わない
4. **例えは最大1つ、1行まで**。例えで世界観を作らない。構造の理解を助けるときだけ使う
5. **1回の出力は1画面以内**（目安: 40行）。足りない分は最後に「深掘りできる場所」を箇条書きで示し、聞かれたら掘る
6. **正確さを犠牲にしない**。単純化で嘘になるなら「厳密には例外がある」と1行添える

## 出力フォーマット

この型を基本にする（トピックの性質で不要な節は削ってよい。増やさない）:

```markdown
## 一言でいうと
（結論1〜2文。これだけ読めば会話についていけるレベル）

## 構造
（階層箇条書き・表・ASCII図のどれか1つ。全体像を1つの絵で）

## これだけ押さえる
（重要ポイントを最大3つ。それぞれ2行以内）

## 用語の噛み砕き
（本文で出た難語を「語: 1行」で。無ければ節ごと削る）

## 次に知りたくなったら
（深掘り候補を2〜3個。「◯◯って聞いて」形式）
```

## やってはいけないこと

- ファイルを作る（チャット出力のみ。保存したいと言われたら technical-note に渡す）
- 網羅しようとする（「全部で6種類ありますが」より「9割はこの2つ」が正しい）
- 教科書の目次順に説明する（相手の質問に直結する順で）
- 確認の質問で始める（まず答える。前提が複数あり得るときは最有力で答えてから「◯◯の場合は変わる」と添える）

## 深掘りループ

ユーザーが「もっと」「◯◯は？」と返してきたら、同じフォーマットでその1点だけを掘る。
1回の深掘りで1トピック。全体を再説明しない。
