# note記事化（note-create）

## 前提

このSkillは ObsidianImus Vault プロジェクトの構造を前提にする：

- **スタイルガイド**: `Company/04_Output/note/_style-guide.md` が存在する
  - なければ「スタイルガイド未作成」エラー、`/structured-workflow` で先に確立を提案
- **入力ソース**: `Vault/01_Notes/` 配下のなぐりがき
- **出力先**: `Company/04_Output/note/` 配下
- **書き込み禁止**: `Vault/` 配下（読むだけ）

---

## ワークフロー（ハイブリッド分岐型・6ステップ）

### Step 1: 入力ソース確認

1. `Vault/01_Notes/` 配下にテーマに関連するなぐりがきがあるか確認
2. **ある場合**: Read で全文取得し、Step 2 へ
3. **ない場合**: 以下を案内：

> Vault/01_Notes/ に該当するなぐりがきが見つかりません。
> 先に自分の言葉で書いてからもう一度呼んでください。
> （Vaultはご主人様の聖域。AIは書き込めません）

### Step 2: 深掘り提案

なぐりがきを読んで以下を提示する：

#### 2-1. 良いところ（既にできている点）
- 構成・視点・自分の言葉で書けている部分を3〜5個

#### 2-2. 足りない視点（深掘り候補）
- 5つ厳選して提案
- 例：時系列の感情の波、価格/投資感の変化、自分の業務への翻訳、失敗・違和感、言語化できなかったこと

#### 2-3. 構造修正の提案
- 章の順序、削除候補、追加候補

#### 2-4. 批判的指摘（あれば）
- 守破離の解釈ミス、論理の飛躍、事実誤認など
- CLAUDE.md「指示を批判的に見る」原則に従う

#### 2-5. 待機
- ご主人様が追記したら再 Read で更新版を取得
- 「追記したよ」コメントを待つ

### Step 3: 方針すり合わせ（毎回必須）

`AskUserQuestion` で以下を確認（最大4問）：

1. **公開範囲** — 実名・固有名詞・金額情報の出し方
2. **トーン** — 砕け度（カジュアル / バランス / フォーマル）
3. **タイトル方向性** — 結論型 / 問い型 / 物語型
4. **特記事項** — 含めたい/避けたい話題

→ 答えを Step 4 のドラフト方針に反映

### Step 4: ドラフト生成

#### 4-1. スタイルガイドを必ず参照
`Company/04_Output/note/_style-guide.md` を Read し、以下を遵守：
- 文末の黄金比率（断定5：推量2以下：疑問1〜2：体言止め2〜3）
- 砕け語の使用上限（マジで3回・ぶっちゃけ2回・ガチで2回）
- 段落2〜4行・見出し20字以内
- 太字は1段落1箇所
- 独自ワード辞書（Claude君など）の活用機会

#### 4-2. 構成テンプレート選択
スタイルガイドの3パターンから選択：
- A. エッセイ型（体験記・人物発信）
- B. 気づき型（短文・1テーマ深掘り）
- C. 連載・シリーズ型（成長記録）

#### 4-3. 出力先
```
Company/04_Output/note/YYYY-MM-DD-タイトル簡略.md
```

frontmatter 必須項目：
```yaml
---
date: YYYY-MM-DD
type: note記事ドラフト
status: 第1稿 / 第2稿 / ...
target: 想定読者
length: 文字数目安
source: Vault/01_Notes/...（参照元）
style_guide: Company/04_Output/note/_style-guide.md
公開可: ...
公開不可: ...
独自ワード: ...
---
```

### Step 5: フィードバックループ

ご主人様の修正指示を反映する。
**大幅変更（タイトル/構成/トーン）は新版**として `_第N稿.md` で保存：
- 第2稿 → トーン調整
- 第3稿 → スタイルガイド準拠での磨き上げ
- 第4稿以降 → 必要に応じて

軽微な修正は同ファイルを Edit で更新。

### Step 6: 公開前チェック

スタイルガイドの「公開前チェックリスト」を実行：

- [ ] タイトル40字以内
- [ ] 冒頭140字に数字あり
- [ ] H2見出し20字以内
- [ ] 1段落3行以内
- [ ] 太字が1段落1箇所以下
- [ ] 砕け語が上限内
- [ ] 文末バランスが黄金比に近い
- [ ] ハッシュタグ5〜7個
- [ ] 登場人物の名前出しを本人に事前共有予定
- [ ] アイキャッチ画像準備
- [ ] X告知文準備
- [ ] 公開時刻設定（朝7時 or 夜21時）

→ 抜け漏れを報告し、ご主人様の最終判断を仰ぐ。

## 重要な注意事項

### 禁止: やってはいけないこと
- `Vault/` 配下に1バイトも書き込まない（読み込みのみ）
- スタイルガイドを無視したドラフト生成
- 確認せずに大幅な構成変更
- 連れていってくれた人・関係者の実名を勝手に出す

### OK: 守るべきこと
- スタイルガイドの**黄金比率**を必ず守る
- 独自ワード辞書（Claude君など）の更新機会があれば提案
- ご主人様の「自分の言葉」を最大限活かす（過度に書き換えない）
- 批判的指摘は遠慮なく（守破離の誤解、論理の飛躍など）
- 連れていってくれた人への配慮（事前共有を必ず提案）

### ヒント: 推奨パターン
- 第3稿まで進めるのが標準（初稿→トーン調整→スタイル磨き上げ）
- 新ワード採用時は「試験投入」として控えめに（1記事1個）
- シリーズ化の余地があれば指摘（連載型テンプレ提案）
