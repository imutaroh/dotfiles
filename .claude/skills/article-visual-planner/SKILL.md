# 記事の図解・挿絵プランナー（article-visual-planner）

記事を読んで、**①どこに・何の図を・何枚入れるべきか（配置プラン）** と **②ChatGPTに渡す画像生成プロンプト一式** を出力する。判断は推測ではなく、`references/best-practices.md` の裏取り済みルール（実務慣行＋学習科学）に従う。

## いつ使うか

- 「この記事に図解／挿絵を入れたい」「画像プロンプトまとめて作って」
- 「どこに図を入れると最適か診断して」「記事に図を最適配置して」
- 書き上げた Zenn / ブログ / 解説記事に、ChatGPT で作る図版を足したいとき
- `/article-visual-planner` で直接呼ばれたとき

## 前提・読み込むもの

作業開始時に必ず以下を読む：

- `references/best-practices.md` — 図版の頻度・配置・アンチパターンの意思決定ルール（このスキルの「脳みそ」）
- `references/image-style.md` — 生成画像のSTYLE仕様とImage-2.0への渡し方、日本語ラベルの入れ方（近接性を画像内で満たす）、出力フォーマット

対象記事は、ファイルパスで渡されたら Read する。貼り付けられたら本文をそのまま使う。

## ワークフロー

### Step 1: 記事を構造で読む
記事を **H2（大見出し）単位** に分解し、各セクションに含まれる「理解の山場」を分類する：
**抽象概念 / 対比（A vs B）/ プロセス・段階 / 構造・関係 / 全体像 / 数値比較 / 操作手順 / 感情の転換点（記事のトーン上重要な箇所）**。

### Step 2: 各候補を「コヒーレンス・ゲート」に通す
各要素に **「この箇所に図が無いと、読者の理解は落ちるか？」** を問う。**YESだけ残す。** 「あったら華やか」程度（＝純装飾・誘惑的詳細）は、ここで落とす。これが最大のアンチパターン回避（`best-practices.md` §4）。

### Step 3: 図の種類を割り当てる（必ず「図解」にする）
残った候補に、`best-practices.md` §3 の対応表で図の種類を割り当てる（対比→左右並置、プロセス→番号付きフロー、構造→関係図、複雑な全体像→big picture＋サブ図、数値→表/チャート、手順→trimしたスクショ）。
**いずれも「説明する図解」にする**：各図に**実際の情報**（サンプルデータ・ラベル付き構造・矢印の流れ・before/after・ステップ）を入れ、図だけ見て概念が分かる状態にする。比喩は容器として使ってよいが、比喩の絵"だけ"の「挿絵」で終わらせない（`best-practices.md` 大原則0）。

### Step 4: 頻度を整える（ノルマ化しない）
- **1図＝最大1段落分／説明に5項目以上要るなら分割**（情報密度ルール）。
- 全体像が複雑なら **big picture 1枚 → サブ図に展開**（progressive disclosure）。
- **「全H2に機械的に1枚」はしない。** 図が無くても通じる節には入れない。
- ⚠️ 効果が最大化するのは「複雑な題材×事前知識の低い読者」。上級者向けの濃い記事なら図を減らす提案をする。

### Step 5: 配置と読み順を設計
- 各図は **「それを説明する段落の直後」** に置く想定で位置を指定（空間的近接性）。
- **1行キャプション**を添える（図が何を表すか）。
- 多ステップ・多要素の図は **番号＋左→右/上→下の視線の流れ**（serial format）。
- 図ごとに **ラベル要否** を判定。ラベルが要る図は、**日本語ラベルを図の該当箇所の真横に直接入れる**指定をSCENEに書く（Image-2.0は日本語描画が高精度＝近接性を画像内で満たせる。後乗せは原則不要）。

### Step 6: プロンプト化
`image-style.md` の STYLE ブロック ＋ 番号付き SCENE 群を生成。**プロンプト・画像内ラベルとも日本語が既定**。対比は左右、プロセスは番号付き、全体像→サブ図は分割。画像内の文字は**指定した日本語ラベルだけ**（装飾文字は入れない）。**最大8枚／1プロンプト**で束ねる。

### Step 7: 出力（チャットに出す。プロンプトは使い捨て）
以下3点をセットで出す：

1. **配置プラン表**：｜位置（H2/段落）｜図の種類｜狙い（符号化する概念）｜根拠原則｜ラベル要否｜
2. **ChatGPT プロンプト一式**：STYLE ＋ 番号付き SCENE（コピペでそのまま使える形）＋ ChatGPTへの渡し方の一文
3. **入れない判断をした箇所**：候補に挙がったが外したもの＋理由（コヒーレンス原則の透明化）

## 原則（毎回守る）

- **枚数を稼がない。1枚＝1目的。** 目的の無い図は提案しない。
- **作るのは「図解（説明する図）」。「挿絵（飾りの比喩絵）」は0枚。** 比喩は容器に使ってよいが、中に実データ・ラベル付き構造・矢印・before/after など実際の情報を必ず入れる。判定＝「図だけ見て概念を説明できるか？」できないなら作り直す。
- 判断の根拠（どの原則・なぜ）を配置プラン表に必ず書く（ブラックボックスにしない）。
- 対象モデルは **Image-2.0（gpt-image-2）**＝日本語テキストを高精度で描く。ラベルが要る図は**画像内に日本語ラベルを直接**入れて近接性を満たす（後乗せは厳密にフォント統一したい時だけのオプション）。プロンプト・ラベルは日本語が既定。
