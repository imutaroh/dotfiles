# 画像スタイル & Image-2.0 への渡し方

> 対象モデル＝**ChatGPT Images 2.0（gpt-image-2、2026年4月リリース）**。日本語を含む多言語テキストを高精度（日本語90〜95%以上）で描画できる。**画像内に日本語ラベルを直接入れてよい**（旧 gpt-image-1 の「日本語が文字化けする」前提は破棄）。1プロンプトから最大8枚生成可。O系列推論で構図・要素数・制約を生成前に検証。
> 絵柄はいむたろ様の好み（手描き感NG・クリーンなフラットインフォグラフィック・画像1枚で完結）に準拠。文字は「装飾では入れない／理解を助ける箇所にだけ入れる」。

## 対象モデルの前提（Image-2.0）

- **日本語テキストは崩れない。** ラベル・見出し・心に残る一言を、日本語で画像内に直接入れてよい。
- これにより、リサーチで最重要だった**空間的近接性（ラベルを図の該当部分の真横に置く）が画像単体で満たせる**＝後乗せ作業は原則不要になった。
- **最大8枚／1プロンプト**。10枚規模でもプロンプト2回で済む。
- 2K解像度・多言語・画像編集（一部だけ直す）も得意。

## プロンプトの言語（既定：日本語）

- **SCENE・STYLE・画像内ラベル、すべて日本語が既定。** ユーザーが読んで調整でき、Image-2.0は日本語を正確に描く。
- 画像内の文字は**「理解を助けるものだけ」**：図の各部ラベル／対比の見出し／心に残る一言。**装飾的・冗長な文字や英数字の羅列は入れない**（コヒーレンス／redundancy原則）。
- まれに構図が崩れる SCENE は、その SCENE だけ英語に切り替えると安定することがある（最後の手段）。

## STYLE ブロック（全画像共通・毎回プロンプト先頭で1回渡す）

```
STYLE（全画像に適用）:
フラットなベクター調のインフォグラフィック・イラスト。クリーン／モダン／ミニマル／幾何学的。
なめらかな図形に控えめなロングシャドウ。手描き感・スケッチ感はなし、3Dレンダリングもなし。
統一カラーパレット：ディープインディゴ／ティール／ウォームアンバー＋少量のソフトコーラル、背景はウォームなオフホワイト(#F4EFE6)。
中央寄せの構図、余白たっぷり、輪郭くっきり、技術ブログ／エディトリアル品質。
文字は「各SCENEで指定した日本語ラベルだけ」を簡潔に入れる。装飾的な文字・英数字の羅列は入れない。細めのサンセリフ。
アスペクト比は16:9。
```

- パレット・アスペクト比は案件ごとに調整可。ただし**1記事内では固定**して統一感を出す。

## Image-2.0 への渡し方（プロンプトは使い捨て・チャット出力でOK）

- Images 2.0 は**1プロンプトで最大8枚**生成できる。10枚なら「STYLE＋SCENE 1〜8」を1回、「SCENE 9〜」を2回目、のように束ねる。
- 絵柄を揃えるため、最初に STYLE を提示し「以降すべてこの STYLE で」と指示。冒頭メッセージ例：

> これから STYLE と複数の「図」を渡します。各図をこの STYLE で生成してください。**画像に文字として描くのは、各図の「表示する文字」に挙げた語だけ**。説明文・図番号・「SCENE」「図N」などのメタ情報は画像に絶対に描かないでください。アスペクト比は16:9。

- 絵柄がブレたら「**前の絵と同じパレット・同じフラットスタイルで**」と添える。
- ラベルの一部が崩れたら、その箇所を指定して「このラベルの文字を『◯◯』に直して」と再生成（Image-2.0 は部分編集が得意）。

## ラベルの入れ方（空間的近接性を画像内で満たす）

- **ラベルが要る図**（フロー図・関係図・5レイヤー図・行/列グリッド・星型スキーマ等）：**各部の日本語ラベルを該当箇所の真横に直接入れる**。これが近接性を最大化し理解を最も上げる。SCENE記述に「各ブロックに『◯◯』と短いラベル」と明記する。
- **比喩イラスト**（レジ・倉庫・湖など）：ラベルは基本不要。必要なら一言だけ。
- 後乗せ（Figma等）は、ブランドフォントで厳密に揃えたい場合のオプション（既定は画像内ラベル）。

## プロンプトの書き方（1図の構造）

**重要（実地で判明）**：Image-2.0 は文字を忠実に描くため、`SCENE 1` などの**構造ラベルやタイトルを画像にそのまま焼き込んでしまう**。これを防ぐ：

- **図番号・タイトルはコードブロックの外**（Markdown 見出し `### 図1：…`）に置き、**ユーザーが貼るコードブロックには番号・見出しを入れない**。
- コードブロックは「描く情景」＋「**表示する文字（これだけ）：** …」の2要素だけにする。
- STYLE と冒頭指示で「**表示する文字に挙げた語だけを描き、それ以外（番号・指示文・タイトル）は描かない**」と毎回釘を刺す。

```
（コードブロックに入れるのはこの2要素だけ）
<描く情景。構図＝左右／放射／左→右の流れ、サンプルデータ、強調点を明示>
表示する文字（これだけ）：「…」「…」   ← 入れる文字が無ければ「なし」
```

- 対比図は「左：… ／ 右：…」。プロセス図は「左→右に番号付き」。全体像→サブ図は分割。
- 各図の狙い（記事のどの概念か・なぜ要るか）は**見出し側**に書く（画像には出さない）。

## 出力フォーマット（このスキルの最終アウトプット）

1. **配置プラン表**：｜位置（H2/段落）｜図の種類｜狙い（符号化する概念）｜根拠原則｜画像内ラベル｜
2. **Image-2.0 プロンプト一式**：STYLE ＋ 番号付き SCENE（各 SCENE にラベル指定）。最大8枚ずつに束ねる
3. **入れない判断をした箇所**：候補に挙がったが「無くても理解が落ちない／純装飾」で外したものを明記（コヒーレンス原則の透明化）
