# コピーライティング・フレームワーク（copywriting-frameworks）

## このSkillの役割

「言いたいことはあるのに、どの順番で書けば読者が動くか分からない」を解決する。
世界標準の説得フレーム **PAS / AIDA / BAB / StoryBrand** から最適な1つを選び（or スタックし）、
ネタをその骨格に流し込んで「読者が最後まで読み、行動する」本文に組み立てる。

> [!warning] 大原則：フレームは「型」であって「中身」ではない
> フレームは順番を保証するだけ。刺さるかどうかは**具体（固有名詞・数字・シーン）**で決まる。
> 抽象的な穴埋めで終わらせず、必ず1ステップに1つは生身のディテールを入れる。
> （cf. Vault の「面白い記事の作り方」＝listicleを捨て1本の物語で実演する）

## 前提（このVaultでの作法）

- **参照元**：`Vault/01_Notes/` `Vault/02_Books/` `Vault/03_Journals/`（読むだけ）
- **出力先**：`Company/Drafts/YYYY-MM-DD-slug.md`（frontmatter は `type: ai-work` / `status` / `created` / `topic` / `promote`）
- **書き込み禁止**：`Vault/` 配下
- **媒体の住み分け**（被らせない）：
  - note の体験談・成長記録 → `note-create` に渡す（このSkillは"構成の骨格"だけ提供）
  - Zenn の技術解説 → `learn`
  - インスタ/リール台本 → `instagram-script`（B-1〜B-5）
  - LP・セールスメール・告知文・X投稿 → **このSkillが主役**

## ワークフロー（5ステップ）

### Step 1: 入力確認（5点を聞く）

`AskUserQuestion` で埋まっていない項目だけ確認する：

1. **ネタ** — 何について書くか（1行）
2. **ゴール** — 読後に読者にしてほしい行動（登録・購入・DM・態度変化）
3. **読者** — 誰に（知識レベル・温度＝Cold/Warm/Hot）
4. **媒体** — LP / メール / X / note / ブログ
5. **温度（語り口）** — 煽る / 落ち着いて信頼 / 物語で

### Step 2: フレーム選定

下の「選定ロジック」で1つ選ぶ。長文なら**スタック**（複数連結）を提案する。

### Step 3: 本文化

選んだフレームのテンプレに沿って、各ステップを本文に起こす。
**各ステップに具体を1つ以上**（数字・固有名詞・シーン）。

### Step 4: 推敲（1文1メッセージ）

- 1文に主張は1つ。接続詞で2文を繋いだら切る
- 抽象語（"効率化""価値"）を具体に置換
- CTAは1つに絞り、動詞で始める（「登録する」「DMを送る」）

### Step 5: 出力

`Company/Drafts/` にドラフト保存。frontmatter に使ったフレーム名を `topic` に明記。

---

## フレーム選定ロジック

| 状況 | 推すフレーム | 理由 |
|---|---|---|
| 読者が問題を自覚している（痛みがある） | **PAS** | 痛みを抉って解決を出す最短ルート |
| 読者がまだ無関心（Cold） | **AIDA** | 注意を引くところから階段で連れていく |
| 変化・ビフォーアフターが主役 | **BAB** | 理想と現実のギャップで動かす |
| ブランド/人を主語にした物語 | **StoryBrand** | 顧客を主人公、自分を導き手に置く |
| 長文（LP・保存版） | **スタック** | 例：AIDAで開く→BABで見せる→PASで閉じる |

## 4つのフレーム（テンプレ）

### PAS（Problem → Agitate → Solve）
痛みを自覚した読者を最短で動かす。

| 段 | 中身 | 1行の指針 |
|---|---|---|
| P | 問題提起 | 読者が「それ俺だ」と思う痛みを名指す |
| A | 煽り・深掘り | 放置するとどうなるかを生々しく見せる |
| S | 解決 | あなたの答えを差し出す（CTA直結） |

### AIDA（Attention → Interest → Desire → Action）
無関心な読者を階段で連れていく。

| 段 | 中身 | 1行の指針 |
|---|---|---|
| A | 注意 | フック1行で立ち止まらせる |
| I | 興味 | 「自分に関係ある」と気づかせる事実 |
| D | 欲求 | 手に入る未来を具体で見せる |
| A | 行動 | 今すぐの一手を1つだけ提示 |

### BAB（Before → After → Bridge）
変化を主役にする。

| 段 | 中身 | 1行の指針 |
|---|---|---|
| B | Before | 現状の不満・閉塞を描く |
| A | After | 変わった後の理想を具体で描く |
| B | Bridge | その橋＝あなたの方法・商品 |

### StoryBrand（SB7・簡易版）
顧客を主人公、自分を導き手にする物語型。

1. 主人公（顧客）が居る
2. 問題に直面する
3. 導き手（あなた）が現れる
4. 計画を授ける
5. 行動を促す
6. 失敗の回避を示す
7. 成功した姿を見せる

## 実演：「新卒はAIをどう使うべきか」を各フレームで（検証用）

> 同じネタが、フレームでどう化けるかの見本。媒体＝X投稿/告知を想定。

**PAS：**
> （P）AIに聞いてから先輩に質問してる新卒、けっこう危ない。
> （A）便利だけど、"なぜそうしたか"を自分で考えないまま納品すると、レビューで詰まって信用を落とす。理由を一度も自分で持ってないから。
> （S）AIは答えの自販機じゃなく壁打ちのサンドバッグ。任せる/任せないの線引きを1枚にまとめた→（リンク）

**AIDA：**
> （A）入社3ヶ月、AIに頼りすぎて先輩に詰められた。
> （I）原因は「AIに考えさせた」こと。「AIと考える」のとは別物だった。
> （D）線引きを変えたら、雑務はAI・判断は自分、で成長速度が逆に上がった。
> （A）その線引きルールはこれ→（リンク）

**BAB：**
> （B）AIの答えをコピペして、自分の頭が育ってない焦り。
> （A）3層モデル（任せる/案だけ/任せない）で線引きしたら、浮いた時間を"なぜ"の理解に回せた。
> （B）橋になったのは、たった1つの判断式：「自分で説明できるか？」

**StoryBrand：**
> 新卒の僕（主人公）はAIに溺れて先輩に詰められた（問題）。
> そこで"3層モデル"という地図（導き手＝考え方）に出会い、
> 任せる/任せないを決め（計画）、毎回"説明できるか"を自問した（行動）。
> おかげで「AIに使われる新卒」になる失敗を避け（回避）、
> AIを相棒に成長を加速させる新卒になれた（成功）。

→ **スタック例（保存版記事）**：AIDAのAで開く → BABのA/Bで変化を見せる → PASのSで線引きを配って締める。

## 禁止 / OK

- **禁止**：抽象的な穴埋めで終える（"問題を提起します"レベル）／CTAを2つ以上盛る／Vault に書き込む
- **OK**：各段に具体を1つ以上／フレームを明示してから書く／長文はスタックを提案／note体験談は note-create に橋渡し
