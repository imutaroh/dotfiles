# Daily Report Formatter

## モード判定（最初に判断する）

| 条件 | モード |
|------|--------|
| チャットにテキストが貼られている（音声文字起こし・話し言葉） | **A: テキスト入力モード** |
| 日付・ファイル名が指定されている、またはテキストがない | **B: 既存ファイルモード** |

---

## モード A：テキスト入力（音声文字起こし → 整形 → 保存）

### 大原則

**整形だけ**を行う。加筆・捏造・肉付け・まとめをしない。話した範囲のまま整える。

### ワークフロー

1. `date +%Y-%m-%d` で今日の日付を取得
2. テキストを5セクションに振り分ける
3. フォーマットに従って整形（加筆しない）
4. `Vault/03_Journals/YYYY-MM-DD.md` に**上書き保存**
5. 保存パスをユーザーに伝える

---

## モード B：既存ファイル（書き殴りメモ → 学びプレビュー → 追記）

### ワークフロー

**Step 1: ファイルを読む**
日付指定があればその日付、なければ前日の日付を `date` コマンドで計算。
`Vault/03_Journals/YYYY-MM-DD.md` を読み込む。

**Step 2: 学び・気づきだけプレビュー提示する（ここで止まって承認を待つ）**

```
### 【確認用】学び・気づき（案）

▼ [見出し]
・[内容]
```

「これでOKなら進めます」と一言添えて承認を待つ。全セクションはまだ書かない。

**Step 3: 承認後に全セクションを整形して追記する**

元ファイルは変更せず、`---` 区切り線の下に追記する：

```
（既存の殴り書きメモ、一切変更しない）

---

【日報】YYYY-MM-DD
...（整形済み日報）
```

モードBでは**詳しく書く**：メモに書いてある内容は省略せず、背景・具体例・フローも含めて丁寧に整形する。

---

## セクション分類ルール（モードA・B共通）

| セクション | 含める内容 |
|-----------|-----------|
| 今日の目標と達成状況 | 目標・達成・未達・補足（背景・経緯） |
| 本日の業務内容 | 実際に行った業務・タスク・ミーティング（箇条書き） |
| 振り返り・自己分析 | 「学び・気づき」▼ブロック。話した分だけ。個数固定しない |
| 先日のFBの返信 | 前回FBへの返答。なければ `（先日のフィードバックなし・割愛）` |
| 明日の目標 | 翌日の目標・ネクストアクション |

---

## ライティングルール

- 箇条書きは `・`（`-` は使わない）
- 語尾は**くだけた敬語**（「〜できました」「〜な気がしてます」。「〜である」調・論文調は使わない）
- 明日の目標のサブ項目は全角スペース2個でインデント
