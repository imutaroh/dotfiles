# Month — 月次振り返りサマリー

## 大原則

- これは **AI が素材（下書き）を出す → いむたろ様が `//` コメントや上書きで自分の言葉にする** ための入力素材。完成品ではない
- Vault 書き込み例外（2026-07-15 いむたろ様承認）。書き込み先は `Vault/03_Journals/Monthly/` のみ
- **既存ファイルの本文（特にいむたろ様の `//` コメント・加筆）は1文字も消さない・書き換えない**
- 事実要約は日報に書いてあることだけ。捏造・肉付けをしない。辛口指摘セクションのみ AI の判断を書いてよい

## ワークフロー

### 1. 対象月の決定

1. `date +%Y-%m-%d` で現在日を取得（推測しない）
2. 引数・会話で月が指定されていればその月
3. 指定がなければ当月。ただし**月初5日以内なら「前月ですか？」と確認**する

### 2. 日報の全読み

1. `ls Vault/03_Journals/ | grep YYYY-MM` で対象月の日報を列挙
2. **全ファイルを Read する**（半分ずつ2バッチの並列 Read）。斜め読み・サンプリングで済ませない
3. 空ファイル・メモ書きだけの日もそのまま受け入れる（無理に埋めない）

### 3. 再放送検出の材料集め

辛口指摘のために、対象月より前の反省・アクションを拾う：

1. 前月の `Monthly/` ノートがあれば Read
2. 無ければ前月日報から反省・アクション行を Grep で拾う（例：`アクション`・`次からは`・`👉`・`◆`）
3. 対象月の日報内で本人が「これは再放送」と自覚している記述（例：6月の「同じ反省の再放送です」）は最優先で引用する

### 4. サマリー生成（チャットに出す）

以下の構成。見出しは日付レンジ付きで具体的に書く：

```markdown
# YYYY年M月の振り返り（AI生成素材）

> [!info]
> このノートは /month スキルが日報から生成した下書き素材。
> いむたろ様が自分の言葉で上書き・注釈（//）して完成させる。

## 実務ベース

（メインプロジェクトを時系列で。「M/D〜M/D フェーズ名：何をして何が決まったか」形式。
サブの動き＝MTG・合宿・LT会・イベント等は箇条書きで別掲）

## 学びベース

### 技術の学び
### 仕事の進め方の学び
### マインドの学び

（各項目に日付と教えてくれた人を残す。日報の言葉をなるべく保つ）

## 辛口指摘

（お世辞なし。最低限これを見る：
- **反省の再放送**：前月以前と同じ反省を繰り返していないか。日付を並べて示す
- **目標の達成状況**：月初・月中に立てた目標が達成されたか。未達なら日報内の原因分析を引用
- **アクションの追跡**：「次からは〜する」と書いたアクションがその後の日報で実行された形跡があるか）

## 来月への検証ポイント

（1〜3個。外部強制（締切・時間指定・他人の目）で検証可能な形になっているものを優先して挙げる）
```

### 5. 保存

保存先: `Vault/03_Journals/Monthly/YYYY-MM-monthly.md`

- **新規の場合**：骨子（見出しだけ）を Write → 各セクションを Edit で肉付け（一発 Write で全文を書かない）
- **既存ファイルがある場合**（`YYYY-MM-monthly.md` でも `6月.md` のような変則名でも）：
  1. まず Read して中身を確認
  2. 本文は一切書き換えず、**末尾に `---` 区切りで「## AI月次サマリー（生成日: YYYY-MM-DD）」として追記**
  3. すでに同じ月の AI サマリーを追記済みなら、上書きせずいむたろ様に確認する
- 保存後、パスをいむたろ様に伝える

## 破綻条件（自覚しておく）

- 日報が書かれていない月に使うと素材ゼロで空振りする → その旨を正直に伝えて終了
- 月の日報が30本を超えたあたりでコンテキストを圧迫する → その場合は前半/後半で Explore サブエージェントに分担させ、結果を自分の言葉で統合する
